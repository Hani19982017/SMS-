# عقار واتساب — نظام التسويق العقاري عبر واتساب 🏠💬

نظام ويب متكامل (SaaS) لإدارة التسويق العقاري عبر **WhatsApp Business Cloud API الرسمي**،
مع تحليل الردود بالذكاء الاصطناعي، تصنيف العملاء، استخراج بيانات العقارات تلقائيًا، ولوحة تحكم عربية كاملة.

---

## ⚙️ التقنيات المستخدمة

| الطبقة | التقنية |
|--------|---------|
| الواجهة | Next.js 14 (App Router) + React + Tailwind CSS (RTL عربي) |
| الخادم | Next.js API Routes (Node.js) |
| قاعدة البيانات | PostgreSQL + Prisma ORM |
| المصادقة | NextAuth.js (أدوار: مدير / مندوب / مشاهد) |
| الذكاء الاصطناعي | OpenAI أو Claude (قابل للتبديل) |
| واتساب | Meta WhatsApp Cloud API (رسمي فقط) |
| الإرسال التدريجي | Render Cron (بديل Redis/BullMQ) |
| النشر | Render |

## 🚀 التشغيل محليًا

```bash
# 1) تثبيت الحزم
npm install

# 2) جهّز ملف البيئة
cp .env.example .env   # ثم عدّل القيم

# 3) أنشئ قاعدة البيانات وطبّق المخطط
npx prisma migrate dev --name init
npx prisma generate

# 4) أنشئ حساب المدير والقالب الافتراضي
npm run db:seed

# 5) شغّل المشروع
npm run dev
```

افتح `http://localhost:3000` وسجّل الدخول ببيانات الـ seed (الافتراضي: `admin@aqar.local` / `Admin@12345`).

## ☁️ النشر على Render

1. ارفع المشروع على GitHub.
2. في Render اختر **New → Blueprint** وأشر إلى `render.yaml`.
3. سيُنشئ تلقائيًا: خدمة الويب + قاعدة PostgreSQL + مهمة Cron للإرسال.
4. املأ المتغيرات السرية (`sync: false`) من لوحة Render:
   - `NEXTAUTH_URL` = رابط تطبيقك على Render.
   - مفاتيح واتساب والذكاء الاصطناعي.
   - في مهمة Cron: `APP_URL` (رابط الويب) و`CRON_SECRET` (نفس قيمته في الويب).
5. بعد أول نشر، شغّل الـ seed مرة واحدة من Shell في Render: `npm run db:seed`.

> ملاحظة: خطة Render المجانية تُنيم الخدمة عند الخمول؛ للإنتاج الجاد استخدم خطة مدفوعة.

## 🔗 ربط Webhook الخاص بواتساب

في لوحة Meta for Developers → WhatsApp → Configuration:

- **Callback URL**: `https://your-app.onrender.com/api/whatsapp/webhook`
- **Verify Token**: نفس قيمة `WHATSAPP_WEBHOOK_VERIFY_TOKEN`
- فعّل الاشتراك في حقول: `messages`.

النظام يتحقق من الرمز تلقائيًا (طلب GET) ويستقبل الرسائل وتحديثات الحالة (طلب POST).

## 🔄 كيف يعمل النظام (التدفق الكامل)

1. **الاستيراد**: ترفع ملف Excel/CSV → النظام يطبّع الأرقام المصرية ويمنع التكرار.
2. **الحملة**: تنشئ حملة بقالب معتمد → تُدرَج الرسائل في طابور بجدولة متدرجة (تحكّم في السرعة).
3. **الإرسال**: مهمة Cron ترسل دفعة كل دقيقة، رسالة واحدة لكل رقم، بفاصل زمني، **فقط للأرقام الموافِقة (opt-in)**.
4. **الرد**: يصل رد العميل عبر Webhook → يُحفظ في المحادثة.
5. **الذكاء الاصطناعي**: يُصنّف العميل (مالك/وسيط/غير مهتم...) ويستخرج بيانات العقار (المنطقة، السعر، الغرف، الفرش...).
6. **الأتمتة**: رد تلقائي مناسب (داخل نافذة 24 ساعة) + تحويل الحالات المهمة لمتابعة يدوية + إنشاء سجل عقار تلقائيًا.
7. **اللوحة**: إحصائيات (مرسل/تسليم/رد/ملاك/عقارات) + بحث + تصدير Excel.

## ⚠️ ملاحظات الالتزام بسياسة واتساب (مهم جدًّا)

- **الرسالة الأولى لأي عميل يجب أن تكون قالبًا معتمدًا (Template) من Meta** — النظام يمنع الإرسال بدون قالب.
- النظام يرسل **فقط لجهات الاتصال التي وافقت (opt-in)**؛ الإرسال البارد لأرقام عشوائية يعرّض رقمك للحظر ويرفع التكلفة.
- الرسائل الحرة (والردود التلقائية) مسموحة **فقط داخل نافذة 24 ساعة** من آخر رسالة واردة — النظام يتحقق من ذلك تلقائيًا.

## 🔧 ما يُنصح بإضافته للتوسّع لاحقًا

- صفحات تفصيلية كاملة (العقارات/المتابعات/الإعدادات بنماذج تحرير) — البنية والـ API جاهزة.
- استبدال Cron بـ **Redis + BullMQ** عند الحجم الكبير (التصميم يدعم ذلك).
- دعم أكثر من رقم واتساب فعليًّا (الجدول `whatsapp_accounts` جاهز).
- استقبال الصور/الملفات من العملاء (حاليًا النص فقط).
- الربط بـ CRM أو موقع عقاري عبر الـ API المنظّم الموجود.
